## What Was Delivered

Thymeleaf dashboard page at `GET /`, resolving the 404 that D3's OAuth2 success handler previously landed on.

**New production files:**
- `src/main/java/.../controller/DashboardController.java` — `@Controller` serving `GET /`; injects `name`, `email`, `provider` from `OAuth2User` principal and `OAuth2AuthenticationToken` into the model; null guard on principal redirects to `/login`
- `src/main/resources/templates/dashboard.html` — user card showing name, email (or "N/A" for Twitter/X), provider badge, logout form (`POST /logout` with Thymeleaf auto-injected CSRF token)

**Test files:**
- `src/test/java/.../controller/DashboardControllerTest.java` — 4 tests: unauthenticated redirect, Facebook user renders correctly, Twitter user shows N/A email, logout button present

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Controller type | `@Controller` (not `@RestController`) | Returns a view name for Thymeleaf rendering |
| Null principal guard | Redirect to `/login` | Spring Security handles this via filter chain but controller-level guard adds safety |
| Twitter email | Display "N/A" | Twitter OAuth2 does not expose email; surfacing "N/A" is clearer than blank |
| Logout | `<form>` POST with CSRF | Spring Security requires POST for logout; Thymeleaf `th:action` auto-injects CSRF token |
